docker学习

  1. image, container，镜像由repository:tag命名，镜像是一个特殊的文件系统，构建后不会被改变，前一层是后一层的基础，会一层层构建。

  2. Dockerfile只是默认的docker构建文件名，执行docker build命令时，docker构建文件可以使用参数指定，不要求必须在当前文件目录下，执行docker build命令时必须指定上下文，例如指定为.时，由于docker采用C/S架构，会将上下文目录下的所有文件打包上传到服务端，在Dockerfile中使用COPY命令可以将上下文目录下的文件移动到镜像下指定目录，注意RUN只能在新镜像内部执行命令。

  3. .dockerignore表示去除不需要作为上下文传递给docker引擎的文件

  4. bootfs包括内核引导器和内核，rootfs则包含了系统中常见的目录和文件，比如/bin，/etc，/proc等等，docker的镜像是使用宿主机的bootfs层的，这使得镜像本身只需要包含rootfs层所需的文件和工具即可，所以镜像占用的存储空间比较少。

  5. 不同linux发行版的rootfs不同，如ubuntu使用apt管理软件，而centos使用yum，但二者的内核层面差别不大。可以在一台宿主机上同时运行不同Linux发行版的镜像而不出错，如在一个安装了centos的宿主机上同时启动Centos镜像的容器和Ubuntu镜像的容器

  6. unionFS联合文件系统，overlay2是联合挂载技术的一种实现，由lower, upper, work三个目录构成，三种目录合并出来的目录称为merged目录

  7. lower目录：作为只读层，可以为多个，处于最底层的目录

  8. upper目录：只有一个，作为读写层

  9. work目录：为工作基础目录，挂载后内容会被清空，在使用过程其内容用户不可见

  10. merged目录：为最后联合挂载完成给用户呈现的统一视图，也就是说merged目录里面本身并没有任何实体文件，给我们展示的只是参与联合挂载的目录里面文件而已，真正的文件还是在ower和upper中

  12. 对于nginx:v2镜像，根据Dockerfile，继承自nginx:lasts镜像，故查看它的LowerDir第一个为/var/lib/docker/overlay2/bf322bf0e883a6275237c925abc767c31d8f3ae7d1cbdbb0237cce98f821dbd3/diff, 即为前一层镜像nginx:lasts的当前层文件（可以理解为nginx:lasts对于它的前一层镜像的修改的UpperDir。对于nginx:v2，它的所有LowerDir都是只读的来源于每一层镜像修改前一层的UpperDir，这样merge之后就得到了当前的视图文件系统

  13. 容器的0号进程为Docker daemon 进程，负责容器中进程的管理；1号进程负责启动业务进程，可以由Dockerfile中的ENTRYPOINT或CMD指定。也可以被 docker run 命令的启动参数所覆盖。
